# Composition docs

Generated by `scripts/compose-graph.py`. To regenerate: `python3 scripts/compose-graph.py`.

## What this is

Auto-derived graphs of how `templates/library/*` workbooks compose. Two views:

- **[`library-graph.md`](library-graph.md)** — top-level cross-template flow. One node per template; edges show which workbooks feed which, with labels describing the data passed.
- **Per-template files** — one per slug, each with a column dependency DAG (which columns feed which inside the workbook) plus the cross-template `Fed by` / `Feeds into` neighbors.

## How to read it

Cross-template edges are inferred from three signals in each `template.json`:

1. A template's `template.description` or `notes` explicitly names another template's slug (e.g. `outbound-3-step-cadence-cold` description names `abm-account-keyed-tier-1`).
2. Free-text upstream phrases — `from upstream <X>`, `upstream email waterfall`, `signal monitor` — imply a known class of upstream template.
3. Free-text downstream phrases — `/clay-outbound`, `ready to hand to` — imply a downstream cadence template.

Edges within a sibling family (e.g. `email-waterfall-us-smb` vs `email-waterfall-eu`) are suppressed — siblings are alternatives, not pipeline steps.

Per-template DAGs come from each column's `depends_on` array plus action `run_condition` formula references.

The auto-generated graph is canonical. The prose flow under `templates/library/INDEX.md` "How templates relate" is editorial and may drift; defer to this.
